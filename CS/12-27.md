1. 가상화가 무엇이고, 이것이 가상머신과 어떠한 차이가 있는지 설명해 주세요.
- 가상화는 하나의 물리 서버 자원을 여러 논리적 실행 환경으로 분리해 사용하는 기술이고, VM은 그 구현 방식으로 하이퍼바이저 위에서 VM마다 Guest OS까지 포함해 완전한 OS 단위 격리를 제공합니다. 그래서 VM은 격리성이 강하지만 OS 오버헤드 때문에 무겁고 느릴 수 있습니다.

2. 그렇다면 Docker는 둘 중 어디에 속하나요? 왜 사람들이 Docker를 많이 채택할까요?
- Docker는 VM처럼 하이퍼바이저로 OS를 통째로 띄우는 방식이 아니라, Host OS 커널을 공유하면서 프로세스를 격리하는 컨테이너 기반(OS 레벨) 가상화에 속합니다. 컨테이너는 Guest OS가 없어 훨씬 가볍고 빨리 실행되며, 이미지로 실행 환경을 표준화할 수 있어 배포/확장/롤백과 CI/CD에 유리해서 널리 채택됩니다.

3. 하나의 Host OS에서 돌아간다면 충분히 한 컨테이너가 다른 컨테이너에 간섭할 수 있는 위험이 있지 않을까요? 이를 어떻게 방어할 수 있을까요?
- 컨테이너는 Host 커널을 공유하므로 설정이 허술하면 간섭이나 탈출 위험이 있습니다. 이를 방어하기 위해 리눅스의 namespace로 실행 환경을 격리하고, cgroups로 CPU/메모리/IO 같은 자원을 제한하며, capabilities/seccomp/AppArmor·SELinux 같은 보안 정책으로 시스템콜과 권한을 최소화합니다. 운영에서는 root 실행 금지, privileged 금지, 불필요한 볼륨 마운트 제한 같은 원칙을 적용합니다.

4. Docker 위에 Docker를 올릴 순 없을까요?
- Docker 위에 Docker를 올리는 것은 가능하며 DinD라고 부릅니다. 컨테이너 내부에 Docker 데몬을 띄우는 방식과, 호스트의 docker.sock을 마운트해 호스트 데몬을 조작하는 방식이 있습니다. 전자는 격리된 도커 환경을 만들 수 있지만 권한 요구가 커지고 복잡하며, 후자는 구현이 간단하지만 docker.sock 접근은 사실상 호스트 제어 권한을 주는 것이라 보안 리스크가 큽니다.
